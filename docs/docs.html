getting started

Install the binary for your platform.

# macOS arm64
curl -Lo wink https://github.com/Malaydewangan09/wink/releases/latest/download/wink-darwin-arm64
chmod +x wink && sudo mv wink /usr/local/bin/

# macOS x86_64
curl -Lo wink https://github.com/Malaydewangan09/wink/releases/latest/download/wink-darwin-x86_64
chmod +x wink && sudo mv wink /usr/local/bin/

# Linux amd64
curl -Lo wink https://github.com/Malaydewangan09/wink/releases/latest/download/wink-linux-amd64
chmod +x wink && sudo mv wink /usr/local/bin/

# from source
go install github.com/Malaydewangan09/wink/cmd/wink@latest

Watch your first process.

wink watch api "node server.js"

Open the TUI to see live logs.

wink

Watch more services. They show up immediately in the running TUI.

wink watch worker "python worker.py"
wink watch db "postgres -D /usr/local/var/postgres"

To stop and clean up.

wink stop api
wink rm api      # removes service + its logs

wink.yaml

Define all your services in a file and start them with one command. Create wink.yaml in any directory.

# wink.yaml
# format: name: command
# lines starting with # are comments

api: go run ./cmd/api
worker: ./bin/worker --env dev
postgres: postgres -D /usr/local/var/postgresql

Then from the same directory.

# start all services
wink up

# start all, pass a different config file
wink up ./config/services.yaml

# stop all services defined in wink.yaml
wink down
Services already running are skipped when you run wink up again. Safe to re-run.

Commands in wink.yaml are passed to sh -c, so shell syntax works.

api: PORT=3000 node server.js
migrate: cd ./db && ./migrate up
redis: redis-server --port 6380

commands

config file

wink up [file]start all services from wink.yaml (default: ./wink.yaml)
wink down [file]stop all services defined in config file

process management

wink watch <name> <cmd>start a process and collect its stdout and stderr
wink stop <name>send SIGTERM to a running service
wink restart <name>stop and restart with the same command, clears old logs
wink rm <name>remove a stopped service and all its logs
wink attach <name> <pid>register an already-running process by PID (no log collection)

viewing

winkopen the live TUI
wink lslist all services, status, and uptime
wink logs [name]print all logs, optionally filtered by service
wink tail [name]follow logs in real time (like tail -f)
wink cleardelete all logs and service state

settings

wink config showprint current config values and file path
wink config editopen config file in $EDITOR
wink config set <key> <val>set a config value
wink config unset <key>reset a config value to its default

TUI controls

Run wink with no arguments to open the TUI. The left pane lists services, the right pane shows logs.

navigation

↑ ↓navigate services (left pane) or scroll logs (right pane)
tabswitch focus between services and logs pane
gjump to oldest logs (top)
Gjump to newest logs (bottom)

filtering

ashow logs from all services
/enter search mode — type to filter logs by keyword
enterapply search and exit search mode (keeps filter active)
escclear search query

display

ttoggle timestamps on/off

service actions

sstop the selected service (sends SIGTERM)
rrestart the selected service (clears old logs)
xremove selected service — press twice to confirm, esc to cancel
qquit wink (does not stop services)

config

Settings live in ~/.wink/config.yaml. The file is created with comments on first use.

# open in your editor
wink config edit

# or set directly
wink config set notify_cmd "terminal-notifier -title wink -message {msg}"
wink config set max_log_lines 5000
key default description
notify_cmd platform default shell command to run on crash. use {msg} as placeholder for the message.
max_log_lines 0 (no limit) cap total lines in logs.jsonl. oldest lines trimmed automatically every 100 appends.

crash notifications

wink sends a notification when a service exits with a non-zero code. Intentional stops via SIGTERM are not treated as crashes.

Platform defaults: osascript on macOS, notify-send on Linux. Override with any tool using notify_cmd.

# terminal-notifier (brew install terminal-notifier)
wink config set notify_cmd "terminal-notifier -title wink -message {msg} -sound Basso"

# alerter
wink config set notify_cmd "alerter -title wink -message {msg}"

# any custom script
wink config set notify_cmd "~/scripts/notify.sh {msg}"

example: node + python worker

A typical web app with an API server and a background job processor.

wink.yaml project root 
api:    node server.js
worker: python -m worker
redis:  redis-server
start everything 
wink up

Open wink, select worker in the left pane to see only its logs. Press / and type error to find failures fast. Press r to restart a service without leaving the TUI.

example: go microservices

Multiple Go services in a monorepo, each in their own directory.

wink.yaml 
gateway:  go run ./cmd/gateway
users:    go run ./cmd/users
orders:   go run ./cmd/orders
postgres: postgres -D /usr/local/var/postgresql

Services share the same log view. When orders panics and crashes, you get a desktop notification and the service shows as dead in the sidebar. Press r to restart it after fixing the code.

watch a single service with env vars 
wink watch gateway "DEBUG=true go run ./cmd/gateway"

example: java backend + react frontend

Spring Boot API alongside a Vite dev server.

wink.yaml 
api:      ./mvnw spring-boot:run
frontend: npm run dev --prefix ./frontend
Spring Boot logs are verbose. Use max_log_lines: 2000 in config to keep logs.jsonl from growing too large.

Java stack traces show up on stderr, which wink renders slightly dimmer to reduce noise. Exception lines are colored red so they stand out.

example: attach to a running process

If a process is already running (started externally or before wink was installed), register it by PID.

attach by pid 
# find the pid
lsof -i :3000

# register it
wink attach api 81234
Attached processes show up in wink ls and the TUI, but wink cannot collect their logs since the pipe was not set up at start.

To get logs from an attached process, stop it, then restart with wink watch.
